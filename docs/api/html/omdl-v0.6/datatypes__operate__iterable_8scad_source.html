80 //! Find the occurrences of a match value in an iterable value.
81 /***************************************************************************//**
82  \param mv <value> A match value.
83  \param v <value> An iterable value.
84  \param c <integer> A match count.
85  For <tt>(c>=1)</tt>, return the first \p c matches.
86  For <tt>(c<=0)</tt>, return all matches.
87  \param i <integer> The element index to consider for iterable elements.
88  \param i1 <integer> The element index where find begins (default: first).
89  \param i2 <integer> The element index where find ends (default: last).
90 
91  \returns <list> A list of indexes where elements match \p mv.
92  Returns \b empty_lst when no element of \p v matches \p mv
93  or when \p v is not iterable.
94 
95  \details
96 
97  The use-cases for find() and [search()] are summarized in the
98  following tables.
99 
100  \b Find:
101 
102  | mv / v | string | list of scalars | list of iterables |
103  |---------------------|:------:|:-----------------:|:-------------------:|
104  | scalar | | (a) | (b) see note 1 |
105  | string | (c) | | (b) see note 1 |
106  | list of scalars | | | (b) see note 1 |
107  | list of iterables | | | (b) see note 1 |
108 
109  \b Search:
110 
111  | mv / v | string | list of scalars | list of iterables |
112  |---------------------|:------:|:-----------------:|:-------------------:|
113  | scalar | | (a) | (b) |
114  | string | (d) | invalid | (e) see note 2 |
115  | list of scalars | | (f) | (g) |
116  | list of iterables | | | (g) |
117 
118  \b Key:
119 
120  \li (a) Identify each element of \p v that equals \p mv.
121  \li (b) Identify each element of \p v where \p mv equals the element at
122  the specified column index, \p i, of each iterable value in \p v.
123  \li (c) If, and only if, \p mv is a single character, identify each
124  character in \p v that equals \p mv.
125  \li (d) For each character of \p mv, identify where it exists in \p v.
126  \b empty_lst is returned for each character of \p mv absent from \p v.
127  \li (e) For each character of \p mv, identify where it exists in \p v
128  either as a numeric value or as a character at the specified column
129  index, \p i.
130  \b empty_lst is returned for each character of \p mv absent from \p v.
131  \li (f) For each scalar of \p mv, identify where it exists in \p v.
132  \b empty_lst is returned for each scalar of \p mv absent from \p v.
133  \li (g) For each element of \p mv, identify where it equals the element
134  at the specified column index, \p i, of each iterable value in \p v.
135  \b empty_lst is returned for each element of \p mv absent from \p v
136  in the specified column index.
137 
138  \note \b 1: When \p i is specified, that element column is compared.
139  Otherwise, the entire element is compared. Functions find()
140  and [search()] behave differently in this regard.
141  \note \b 2: Invalid use combination when any element of \p v is a
142  string. However, an element that is a list of one or more
143  strings is valid. In which case, only the first character of
144  each string element is considered.
145 
146  [search()]: https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Other_Language_Features#Search
147 *******************************************************************************/
148 function find
149 (
150  mv,
151  v,
152  c = 1,
153  i,
154  i1 = 0,
155  i2
156 ) = !is_iterable(v) ? empty_lst
157  : (i1 > i2) ? empty_lst
158  : (i1 > len(v)-1) ? empty_lst
159  : ((not_defined(i) && (v[i1] == mv)) || (v[i1][i] == mv)) ?
160  (
161  (c == 1) ? [i1]
162  : concat(i1, find(mv, v, c-1, i, i1+1, i2))
163  )
164  : find(mv, v, c, i, i1+1, i2);
